# BitPerfect 需求分析文档

> 版本：v1.1 | 日期：2026-07-07 | 阶段：第一阶段 — 需求与规划

---

## 一、产品定位

### 1.1 一句话定位

**BitPerfect** 是一款跨平台本地音乐播放器，核心价值是绕过操作系统音频重采样，以独占模式（Hog Mode / WASAPI Exclusive / ALSA 直通）将原始音频数据"比特完美"地输出到 DAC，确保听感与录音室一致。

### 1.2 要解决的问题

- macOS / Windows / Linux 系统默认会对音频做**采样率转换和音量混音处理**，导致音质损失
- 市面现有播放器（如 Audirvana、JRiver）要么**收费昂贵**，要么交互复杂，普通用户上手困难
- 缺少一个**兼顾易用性与专业音质**的开源方案

### 1.3 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| **Hi-Fi 发烧友** | 拥有外置 DAC，关注采样率、码率、Integer Mode 等专业参数 | 确保 bit-perfect 数据链路，可查看当前音频链路状态 |
| **普通音乐爱好者** | 有本地音乐文件（FLAC/MP3 等），希望比系统自带播放器音质更好 | 界面美观、操作简单，默认设置就能获得最佳音质，无需懂专业术语 |

**设计原则**：默认简单易用，高级功能通过"设置 → 音频"面板开放，满足两类用户。

## 二、功能需求（按优先级分层）

> **设计原则**：优先验证核心技术风险（绕过系统 SRC），再逐步叠加 UI 和高级功能。每一层都可独立交付验证，避免中后期因核心不可行而推倒重来。

### P0 —— 核心验证（最高优先级，最先完成）

**目标**：用最少的代码证明"绕过系统 SRC、bit-perfect 直出音频到 DAC"在技术选型下完全可行。需验证两条数据路径：Float 模式（默认）和 Integer 模式（跳过 CoreAudio Float32 转换层）。

**交付物**：一个命令行测试程序，输入音频文件路径，自动配置 Hog Mode + 采样率匹配，依次测试 Float 和 Integer 两条路径，将 PCM 数据直出到 DAC，并打印当前音频链路状态（文件采样率 / DAC 采样率 / 是否独占 / 数据格式 Float 或 Integer），用于人工确认 SRC 已绕过、Integer Mode 是否可用。

#### 2.1 基础音频文件解码

- 支持无损格式：**FLAC、WAV**（P0 阶段仅这两种，验证链路即可）
- 解码后的 PCM 数据直接送入播放缓冲区，不做任何 DSP 处理（采样率转换、增益调整等全部跳过）

#### 2.2 独占模式音频输出（macOS Hog Mode）

- 通过 CoreAudio 获取音频设备列表，识别支持 Hog Mode 的 DAC
- 调用 `AudioDeviceSetProperty` 以 `kAudioDevicePropertyHogMode` 独占设备
- 独占期间阻止系统和其他应用的声音输出到该 DAC
- 播放结束 / 程序退出时自动释放 Hog Mode

#### 2.3 采样率自动切换

- 读取音频文件的原始采样率（如 44.1kHz / 48kHz / 88.2kHz / 96kHz / 176.4kHz / 192kHz / 352.8kHz / 384kHz）
- 播放前自动将 DAC 切换到匹配的采样率（通过 `AudioDeviceSetProperty` 设置 `kAudioDevicePropertyNominalSampleRate`）
- 采样率切换时输出静音，避免爆音

#### 2.4 Integer Mode 直通

- 绕过 CoreAudio 的 Float32 转换层，直接以整数格式（如 Int16 / Int24 / Int32）向 DAC 发送数据
- 通过 `kAudioStreamPropertyPhysicalFormats` 获取 DAC 支持的物理格式列表
- 匹配音频文件的位深（16bit / 24bit / 32bit）与 DAC 支持的整数格式，设置最合适的输出格式
- 若 DAC 不支持 Integer Mode，CLI 程序打印提示并仅验证 Float 路径（不阻塞 P0 验证）

#### 2.5 命令行测试程序

- 命令行参数：指定音频文件路径、目标 DAC 设备名
- 播放完整曲目，实时打印播放进度
- 启动时依次测试 Float 路径和 Integer 路径，打印每条路径的音频链路信息（采样率 / 是否独占 / 数据格式），供人工确认 SRC 已绕过
- 无 GUI，纯命令行交互

> **P0 不包含**：UI 界面、数据库、播放列表、封面显示、ALAC/AIFF/MP3/AAC 等额外格式。这些在 P0 验证通过后再做。

---

### P1 —— 基础播放器 MVP（P0 验证通过后开始）

**前置条件**：P0 核心链路已验证可行。

**目标**：在已验证的音频链路上加 UI 壳，成为一个"能用"的本地音乐播放器。

#### 2.6 完整格式支持

- 在 P0 的 FLAC/WAV 基础上，追加支持：**ALAC、AIFF、MP3、AAC**
- > ⚠️ DSD 格式（DSF/DFF）暂不支持。DSD 用户群体过于小众，待应用主体开发完毕后视情况再考虑

#### 2.7 播放控制核心

- **播放 / 暂停**：从当前曲目开始或暂停位置播放
- **上一首 / 下一首**：切换到播放列表中上一首或下一首
- **进度拖动**：可拖动进度条跳转到任意位置
- **音量控制**：软件音量（PCM 层面调整），可选是否启用
  - *设计意图*：软件音量通过修改 PCM 数据实现，会降低有效位深，破坏 bit-perfect 链路。默认启用（便利普通用户），发烧友可关闭以保持数字域零修改，改用 DAC/功放模拟旋钮控制音量
- **播放模式**：顺序播放、单曲循环、列表循环、随机播放

#### 2.8 基础 UI 界面

- 主窗口包含：封面展示区、播放进度条、播放控制按钮、当前曲目信息
- 播放列表侧边栏 / 底部面板（当前播放队列，拖拽添加文件）
- 菜单栏基础菜单（文件、播放控制、帮助）
- **界面语言**：界面文字和操作提示以简体中文为主（按钮标签、菜单项、提示信息、设置选项等均为中文显示），首次实现即遵循此原则

---

### P2 —— 音乐管理（第三阶段完成）

**目标**：完整的音乐库管理能力，让用户可以浏览、搜索、管理本地音乐收藏。

#### 2.9 音乐库扫描与元数据解析

- 用户指定本地音乐文件夹（可多个），应用扫描其中所有支持的音频文件
- 解析元数据标签（ID3v1 / ID3v2 / Vorbis Comment / APE tag）
- 提取：歌曲名、艺术家、专辑名、专辑艺术家、音轨号、年份、流派、封面图
- 按艺术家 → 专辑分层浏览
- 按文件夹目录结构浏览（基础视图）

#### 2.10 播放列表管理

- 创建 / 删除 / 重命名播放列表
- 拖拽添加歌曲到播放列表
- 从播放列表中移除歌曲
- 拖动排序
- 播放列表导入 / 导出（M3U 格式）

#### 2.11 封面显示

- 优先读取音频文件内嵌封面
- 回退读取文件夹内的 `cover.jpg` / `cover.png` / `folder.jpg`
- 在主窗口中显示封面（带圆角阴影）

#### 2.12 全文搜索

- 支持按歌曲名、艺术家、专辑名搜索（中文 + 英文混合）
- 搜索结果实时展示，点击即播放

---

### P3 —— 体验增强（第四阶段完成）

**目标**：面向发烧友的高级设置和视觉打磨。

#### 2.13 高级设置面板

- **音频设备选择器**：列出所有可用 DAC，显示设备名称、支持的最大采样率 / 位深
- **Hog Mode 开关**：允许用户手动开启 / 关闭独占模式
- **Integer Mode 开关**：允许手动开启 / 关闭 Integer Mode
- **音频链路状态显示**：实时展示"文件采样率 → DAC 当前采样率 → 输出模式（独占/共享）→ 数据格式（Float/Integer）"
- **缓冲区大小设置**：允许调整音频缓冲区大小（影响延迟和稳定性）

#### 2.14 界面主题

- 浅色主题 / 深色主题
- 跟随系统主题自动切换

#### 2.15 全局快捷键

- 播放/暂停、上一首/下一首、音量增/减 的全局快捷键
- 支持媒体键（键盘上的播放/暂停专用按键）

#### 2.16 菜单栏迷你控制器

- 菜单栏图标，点击展开迷你播放控制器
- 显示当前歌曲名 + 播放/暂停/切歌按钮
- 点击可恢复主窗口

---

### P4 —— 未来扩展（长期规划，暂不实现）

#### 2.17 跨平台支持

- Windows：WASAPI Exclusive Mode
- Linux：ALSA 直通（`hw:` 设备）

#### 2.18 高级功能

- **UPnP / DLNA 投放**：将音频投放至网络播放设备
- **均衡器（EQ）**：可选图形均衡器（⚠️ 与 bit-perfect 理念冲突，默认关闭，作为可选项）
- **ReplayGain**：音量标准化（播放列表内不同曲目的音量统一）
- **CUE 分轨**：支持 CUE Sheet 文件，将一个整轨文件拆分为独立曲目显示和播放
- **音频格式转换**：导出为其他格式（如 FLAC → ALAC）
- **DSD 支持**：DSF/DFF 格式解码与播放

## 三、非功能需求

### 3.1 音质保证

- **Bit-Perfect 数据路径**：从文件解码到 DAC 输出，期间不经过任何采样率转换、位深转换（除非 DAC 不支持）、音量混音、DSP 效果处理
- **可验证性**：提供音频链路状态面板，用户可确认当前数据路径是否为 bit-perfect
- **无爆音/杂音**：采样率切换、播放启停时通过静音过渡避免可闻爆音

### 3.2 性能

- **CPU 占用**：空闲时 < 1%，播放时 < 5%（Apple Silicon 基准，播放 44.1kHz/16bit FLAC）
- **内存占用**：基础运行 < 100MB，加载 10,000 首曲库 < 500MB
- **启动时间**：冷启动 < 3 秒
- **曲库扫描速度**：10,000 首歌曲首次扫描 < 60 秒，增量扫描 < 5 秒

### 3.3 兼容性

- **操作系统**：macOS 13（Ventura）及以上版本
- **芯片架构**：Apple Silicon（ARM64）原生支持，Intel（x86_64）兼容
- **DAC 设备**：测试主流 USB DAC（Topping、SMSL、Schiit、RME、Focusrite、AudioQuest 等品牌各至少一款）

### 3.4 可用性

- **新手友好**：首次启动引导用户选择音乐文件夹，自动扫描，开箱即用
- **界面清晰**：主要功能一键可达，高级设置不影响日常使用
- **错误处理**：DAC 断开、文件不存在、格式不支持等场景有明确提示

### 3.5 稳定性

- 连续播放 24 小时无崩溃、无内存泄漏
- 快速切歌（< 1 秒间隔）不发生竞态条件崩溃
- DAC 热插拔（拔出 / 插入 / 更换）时优雅降级和恢复

## 四、约束与假设

| 类别 | 内容 |
|------|------|
| **平台** | 当前仅开发 macOS 版本，跨平台架构预留扩展点 |
| **音频来源** | 仅本地文件播放，不支持流媒体（Tidal/Qobuz 等） |
| **音频处理** | 不添加任何 DSP 效果器，保持原始数据路径 |
| **技术栈** | 音频核心使用 C++，UI 框架待技术选型确定 |
| **构建系统** | 待技术选型确定（CMake 为默认候选） |
| **开源协议** | 待确定 |
