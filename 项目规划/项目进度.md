# BitPerfect 项目进度文档

> 跨平台本地音乐播放器，支持直通（独占）模式播放，绕过系统重采样。
> 当前阶段：macOS 优先开发。

---

## 第一阶段：需求与规划

### 1.1 需求分析 ✅

> 详细文档：[需求分析.md](需求分析.md)

已完成需求分析，核心要点：

- **产品定位**：跨平台本地音乐播放器，绕过系统重采样，以独占模式直出音频到 DAC
- **目标用户**：Hi-Fi 发烧友（默认简单，高级功能可展开）+ 普通音乐爱好者（开箱即用）
- **功能分层**：P0（核心验证，4 项）→ P1（基础播放器 MVP，4 项）→ P2（音乐管理，4 项）→ P3（体验增强，4 项）→ P4（未来扩展，2 项）
- **非功能需求**：bit-perfect 数据路径验证、CPU < 5%、内存 < 100MB、冷启动 < 3 秒
- **平台**：macOS 13+，Apple Silicon 原生 + Intel 兼容
- **格式支持**：FLAC、WAV、ALAC、AIFF、MP3、AAC（DSD 暂不支持）

### 1.2 竞品调研 ✅

> 详细文档：[竞品调研.md](竞品调研.md)

已完成 6 款主流播放器调研（Audirvāna / JRiver / Foobar2000 / Pine Player / Roon / Colibri），核心结论：

- **BitPerfect 最大机会**：跨平台统一现代 UI + 易用 + bit-perfect 的交集地带，目前无竞品做好
- **差异化方向**：双模式（新手/发烧友）、音频链路可视化、开源可审计、买断制低价
- **竞品空白**：菜单栏迷你控制器仅 Colibri 支持；音频链路可视化仅 Roon/Audirvāna 有

### 1.4 架构设计 ✅

> 详细文档：[架构设计.md](架构设计.md)

已完成四层架构设计，核心要点：

- **分层架构**：UI 层 → 应用层 → 领域层 → 基础设施层，依赖严格单向
- **18 个模块**：基础设施层 5 个（Logger/ConfigStore/Database/PlatformUtils/Threading）、领域层 6 个（AudioEngine/AudioDecoder/Transport/MusicLibrary/Playlist/Metadata）、应用层 5 个（AppController/PlayerController/LibraryController/PlaylistController/SettingsManager）、UI 层 7 个

- **音频数据流**：磁盘文件 → AudioDecoder(后台线程) → AbstractFifo(无锁缓冲) → AudioEngine(音频线程) → CoreAudio → DAC
- **P0→P1→P2→P3 扩展路径**：每阶段新增文件已明确，扩展点已预留（Integer Mode、主题系统、设置面板）
- **数据库 Schema**：tracks + tracks_fts + playlists + playlist_tracks + schema_version，共 5 张表
- **目录结构**：60+ 文件，完整项目骨架

### 1.6 C++ 编码规范 ✅

> 详细文档：[C++编码规范.md](C++编码规范.md)

已完成 C++ 编码规范制定，共 13 章，核心要点：

- **禁用 auto**：所有变量必须显式声明类型，禁止任何类型推导语法
- **前置返回类型**：禁止 trailing return type，与 Java 风格保持一致
- **命名规范**：类 PascalCase、函数 camelCase、变量 snake_case、成员变量 snake_case + 尾部 `_`（Google 规范）、文件/目录 snake_case
- **类型规范**：`nullptr`、`using` 别名、`enum class` 强类型枚举、禁止 C 风格转换
- **内存管理**：智能指针优先、裸指针仅用于不持有所有权的场景、禁止裸 new/delete
- **音频线程安全**：硬实时约束，回调中禁止锁/分配/I/O/日志/系统 API
- **JUCE 规范**：优先使用 JUCE 封装、Listener 模式、跨线程 UI 更新用 callAsync()
- **注释规范**：中文注释、文件头 + 类 + 方法 JavaDoc 风格、关键逻辑逐行注释
- **格式规范**：K&R 大括号、4 空格缩进、必须使用大括号（即使单行）
- **CMake 规范**：源文件显式列出、`BP_` 前缀项目级变量
- **测试规范**：Catch2、`test_<模块>.cpp`、公开接口全覆盖

### 2.2 基础音频文件解码（FLAC/WAV，PCM float）

> **目标**：实现 AudioDecoder 模块，将 FLAC/WAV 文件解码为 PCM float 数据，放入 AbstractFifo 无锁环形缓冲区，供后续音频线程消费。
> **不涉及**：音频设备输出（2.3）、采样率切换（2.4）、Integer Mode（2.5）。
>
> **调用关系**：（CLI）→ `open()` → `startDecoding()` → `decodingLoop()` → AbstractFifo → （2.3 音频引擎消费）。
>
> **架构优化说明**（相比架构设计文档）：
> 1. 去掉 `juce::AudioFormatReaderSource` —— 纯解码场景直接用 `AudioFormatReader::read()`，无需 `AudioTransportSource` 的配套类
> 2. 去掉 `ready_buffer_` —— `AbstractFifo` 本身就是环形缓冲区，无需额外缓冲
> 3. `AbstractFifo` 大小动态计算 —— 在 `open()` 中根据采样率×声道数×0.5 秒动态分配，而非写死常量
> 4. P0 阶段 Listener 通知留空 —— 无 UI 层消费者，P1 补全 `MessageManager::callAsync` 通知逻辑
