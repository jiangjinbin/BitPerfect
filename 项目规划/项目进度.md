# BitPerfect 项目进度文档

> 跨平台本地音乐播放器，支持直通（独占）模式播放，绕过系统重采样。
> 当前阶段：macOS 优先开发。

---

## 第一阶段：需求与规划

### 1.1 需求分析 ✅

> 详细文档：[需求分析.md](需求分析.md)

已完成需求分析，核心要点：

- **产品定位**：跨平台本地音乐播放器，绕过系统重采样，以独占模式直出音频到 DAC
- **目标用户**：Hi-Fi 发烧友（默认简单，高级功能可展开）+ 普通音乐爱好者（开箱即用）
- **功能分层**：P0（MVP 必备，6 项）→ P1（核心体验，4 项）→ P2（体验增强，4 项）→ P3（未来扩展，2 项）
- **非功能需求**：bit-perfect 数据路径验证、CPU < 5%、内存 < 100MB、冷启动 < 3 秒
- **平台**：macOS 13+，Apple Silicon 原生 + Intel 兼容
- **格式支持**：FLAC、WAV、ALAC、AIFF、MP3、AAC（DSD 暂不支持）

### 1.2 竞品调研 ✅

> 详细文档：[竞品调研.md](竞品调研.md)

已完成 6 款主流播放器调研（Audirvāna / JRiver / Foobar2000 / Pine Player / Roon / Colibri），核心结论：

- **BitPerfect 最大机会**：跨平台统一现代 UI + 易用 + bit-perfect 的交集地带，目前无竞品做好
- **差异化方向**：双模式（新手/发烧友）、音频链路可视化、开源可审计、买断制低价
- **竞品空白**：菜单栏迷你控制器仅 Colibri 支持；音频链路可视化仅 Roon/Audirvāna 有

### 1.3 技术选型 ✅

### 1.4 架构设计 ✅

> 详细文档：[架构设计.md](架构设计.md)

已完成四层架构设计，核心要点：

- **分层架构**：UI 层 → 应用层 → 领域层 → 基础设施层，依赖严格单向
- **18 个模块**：基础设施层 5 个（Logger/ConfigStore/Database/PlatformUtils/Threading）、领域层 6 个（AudioEngine/AudioDecoder/Transport/MusicLibrary/Playlist/Metadata）、应用层 5 个（AppController/PlayerController/LibraryController/PlaylistController/SettingsManager）、UI 层 7 个
- **线程模型**：3 类线程（音频实时线程/UI 主线程/后台线程池），音频线程硬实时约束（无锁/无分配/无 I/O），通过 AbstractFifo 无锁通信
- **音频数据流**：磁盘文件 → AudioDecoder(后台线程) → AbstractFifo(无锁缓冲) → AudioEngine(音频线程) → CoreAudio → DAC
- **P0→P1→P2 扩展路径**：每阶段新增文件已明确，扩展点已预留（Integer Mode、主题系统、设置面板）
- **数据库 Schema**：tracks + tracks_fts + playlists + playlist_tracks + schema_version，共 5 张表
- **目录结构**：60+ 文件，完整项目骨架
