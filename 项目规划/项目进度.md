# BitPerfect 项目进度文档

> 跨平台本地音乐播放器，支持直通（独占）模式播放，绕过系统重采样。
> 当前阶段：macOS 优先开发。

## 第一阶段：需求与规划

### 1.1 需求分析 ✅

> 详细文档：[需求分析.md](需求分析.md)

已完成需求分析，核心要点：

- **产品定位**：跨平台本地音乐播放器，绕过系统重采样，以独占模式直出音频到 DAC
- **目标用户**：Hi-Fi 发烧友（默认简单，高级功能可展开）+ 普通音乐爱好者（开箱即用）
- **功能分层**：P0（核心验证，4 项）→ P1（基础播放器 MVP，4 项）→ P2（音乐管理，4 项）→ P3（体验增强，4 项）→ P4（未来扩展，2 项）
- **非功能需求**：bit-perfect 数据路径验证、CPU < 5%、内存 < 100MB、冷启动 < 3 秒
- **平台**：macOS 13+，Apple Silicon 原生 + Intel 兼容
- **格式支持**：FLAC、WAV、ALAC、AIFF、MP3、AAC（DSD 暂不支持）

### 1.2 竞品调研 ✅

> 详细文档：[竞品调研.md](竞品调研.md)

已完成 6 款主流播放器调研（Audirvāna / JRiver / Foobar2000 / Pine Player / Roon / Colibri），核心结论：

- **BitPerfect 最大机会**：跨平台统一现代 UI + 易用 + bit-perfect 的交集地带，目前无竞品做好
- **差异化方向**：双模式（新手/发烧友）、音频链路可视化、开源可审计、买断制低价
- **竞品空白**：菜单栏迷你控制器仅 Colibri 支持；音频链路可视化仅 Roon/Audirvāna 有

### 2.2 基础音频文件解码（FLAC/WAV，PCM float）

> **目标**：实现 AudioDecoder 模块，将 FLAC/WAV 文件解码为 PCM float 数据，放入 AbstractFifo 无锁环形缓冲区，供后续音频线程消费。
> **不涉及**：音频设备输出（2.3）、采样率切换（2.4）、Integer Mode（2.5）。
>
> **调用关系**：（CLI）→ `open()` → `startDecoding()` → `decodingLoop()` → AbstractFifo → （2.3 音频引擎消费）。
>
> **架构优化说明**（相比架构设计文档）：
> 1. 去掉 `juce::AudioFormatReaderSource` —— 纯解码场景直接用 `AudioFormatReader::read()`，无需 `AudioTransportSource` 的配套类
> 2. 去掉 `ready_buffer_` —— `AbstractFifo` 本身就是环形缓冲区，无需额外缓冲
> 3. `AbstractFifo` 大小动态计算 —— 在 `open()` 中根据采样率×0.5 秒动态分配（管理单声道读写进度），而非写死常量；`fifo_buffer_` 与 `decode_buffer_` 类型统一为 `juce::AudioBuffer<float>`，拷贝无需格式转换
> 4. P0 阶段 Listener 通知留空 —— 无 UI 层消费者，P1 补全 `MessageManager::callAsync` 通知逻辑

### 2.3 音频设备管理（AudioDeviceManager）

> **目标**：封装 CoreAudio 设备操作，实现设备枚举、Hog Mode 独占控制、采样率自动切换、物理格式查询与匹配。所有 CoreAudio API 调用在 UI 线程执行，禁止在音频回调中调用。
>
> 子步骤（待细化）：
> - 设备枚举：获取输出设备列表，识别各设备能力（采样率范围、位深、NonMixable 支持）
> - Hog Mode：通过 `kAudioDevicePropertyHogMode` 独占/释放设备
> - 采样率切换：通过 `kAudioDevicePropertyNominalSampleRate` 匹配源文件采样率
> - 物理格式匹配：查询 `kAudioStreamPropertyAvailablePhysicalFormats`，按源文件属性 + Hog Mode 开关选择目标格式（NonMixable / Mixable）

### 2.4 音频引擎与实时回调（AudioEngine）

> **目标**：实现 JUCE `AudioIODeviceCallback`，在音频实时线程中消费 AudioDecoder 输出的 Planar Float 数据，完成 Planar→Interleaved 转换后写入 HAL 缓冲区。此步骤先实现 Float 路径（Path A），验证基本音频输出链路。
> **前置依赖**：2.3（设备已配置好格式和采样率）
>
> 子步骤（待细化）：
> - `AudioIODeviceCallback` 实现：实现 `audioDeviceIOCallbackWithContext()`
> - Float 路径转换：Planar Float → Interleave → 写入输出缓冲区
> - 与 AudioDecoder 的 Fifo 对接：从 `AbstractFifo` 无锁读取解码数据
> - 静音/淡入处理：采样率切换时输出静音，避免爆音

### 2.5 Integer Mode 直通

> **目标**：在 Float 路径验证通过后，实现 Integer 路径（Path B），达到全程零浮点的 bit-perfect 播放。需同时修改 AudioDecoder（新增 Int32 解码输出）和 AudioEngine（新增 Int32 版 Planar→Interleaved 转换）。
> **前置依赖**：2.3（设备格式匹配）+ 2.4（音频回调框架）
>
> 子步骤（待细化）：
> - AudioDecoder Int32 解码扩展：新增方法调用 `reader->read(int**, ...)` 输出 Planar Int32
> - 位深匹配转换：int32 → 目标位深（24bit/16bit）缩减
> - Planar Int32 → Interleaved Integer 实时转换
> - NonMixable 验证：配合 SMSL DAC 确认 bit-perfect 数据路径
> - 回退逻辑：设备不支持 Integer 格式时自动回退 Float 路径（如 Mac mini 扬声器）
